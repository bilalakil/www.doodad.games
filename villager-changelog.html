Villager Changelog

v0.2.8

  • Fixed camera spasms.
  • Upgraded Unity to 2019.1.10f1.

v0.2.7

New Item Buying System

  • You can now buy items of a specific rarity, starting at 30/90/270/810 gold
  • Each time you buy an item, the buy price for that rarity increases in an exponential manner
  • Each time you sell an item, as well as receiving gold for selling it, it partially reverses the exponential price increase for buying an item of the same rarity
    • Specifically, selling an item reverses 0.4 of the purchase of an item

New/Reworked Features

  • (UI) You can opt in (in the menu) to skip confirmation for some game features:
    • Giving an item to a hero
    • Giving juice to a hero
    • Selecting a target for a hero ability (like Forge)
  • (UI) You can now drag items down from the inventory to sell them quickly
  • (UI) If you drag an item from one hero to another with a full inventory, a swap button will appear on the destination hero's item popups

Balance Changes

  • Base item find chance reduced from 0.25% to 0.225%
  • Item sell amounts increased to 10/30/90/270 gold

Bug Fixes

  • You can no longer place heroes that you can't afford (not sure when this started happening)
  • Treasure and Cache monster abilities now respect their item rarity guarantees
  • Aura items effectiveness is now based on the level of the item holder, instead of the affected heroes

Technical/Other Changes

  • 3D model weapons have been scaled up almost across the board
  • Projectiles have been scaled up almost across the board
  • 3D animations have been tweaked and elongated to be more visible
  • Floating text is now aggregated where applicable - for instance when killing more than one monster at once
  • Many, many foundational code changes:
    • Moved most gameplay formula calculations (i.e. cooldowns, damage amounts, etc.) into one place so they can be maintained with more consistency
    • Reworked the code behind all heroes to make them compatible with a loading system (at the moment all heroes are loaded all the time, which isn't scalable)
    • Reworked all hero abilities to utilise a new, separate targeting and target validity system, needed to support:
      • effects (i.e. from items) that share or depend on a hero's target; and
      • changing of targets mid-animation for attacks/spells, which became necessary with the newly elongated animations
    • Slowly switching to Unity's improved TextMeshPro for text display
